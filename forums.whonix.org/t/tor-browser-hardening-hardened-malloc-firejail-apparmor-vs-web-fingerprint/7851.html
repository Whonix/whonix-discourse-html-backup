Btw I went with firejail, because:

  • /etc/firejail/torbrowser-launcher.profile profile looks simple
  • torbrowser-launcher apparently uses it by default if firejail is installed
  • I didn’t have to write a profile.
  • Profile is maintained upstream https://github.com/netblue30/firejail/blob/master/etc/torbrowser-launcher.profile.
  • Upstream is helping with Tor / Tor Browser related inquiries ( https://github.com/netblue30/firejail/issues?utf8=✓&q=Tor+in%3Atitle ).
  • superb developer support ( https://github.com/netblue30/firejail/issues/2863 )
  • Was easy to implement.
  • No traction / volunteers for anything similar to firejail lately.

A combination is unnecessary at best. It would likely degrade security. bubblewrap is meant to be a replacement.
