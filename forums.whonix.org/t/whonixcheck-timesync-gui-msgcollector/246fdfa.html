It seems we are trying to accomplish something original (in the sense that it was never tried before). In order to stay int acceptable limits, we’ll have to accept a compromise. The solution to rewrite a word wrapping function (either in bash in msgcollector or in python in a external function) is ugly and will never manage all the possible situations. The non-documented solutions on stackexchange just mention some improbable hacks in the C++ code in QT.

The first option is in a new branch: https://github.com/troubadoour/msgcollector/tree/textbrowser.

If all the tests results are OK, the window is displayed with a fixed height, not changeable programmatically. For whonixcheck, because the text is too long for the text area, a scroll bar is added. The timesync window is the same height, without scroll bar. The user can resize (expand only) or maximize the dialogs.

If any test does not pass, the height of the window is set at 1/2 screen height (we can change that, but that will determine the minimum height of the window). The scroll bar is shown if needed.

Other options if warning or error:

  • change the background color
  • scroll to the line and highlight the pretty_type (select). If the warning-error is outside the displayed area, it will be shown at the bottom.

We could use any combinations of the options.
