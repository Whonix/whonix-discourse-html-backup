Patrick August 3, 2016, 8:30pm #7

Still using Whonix 12 is the problem. It most likely upgraded Tor from deb.torproject.org automatically. (Which we no longer do in Whonix 13.)

Upgrading to Whonix 13 however would not fix this issue. [It would disable deb.torproject.org but not downgrade the Tor package.]

I’ve reported a bug against Tor.
/var/run/tor/control socket not created because of /var/run/tor permission issue
https://trac.torproject.org/projects/tor/ticket/19824

Patrick August 3, 2016, 10:34pm #8

No more debug output required!

Bug fixed in git:

github.com/Whonix/anon-gw-anonymizer-config

update /usr/share/tor/tor-service-defaults-torrc 

For compatibility with Tor version 0.2.8.6-1~d80.jessie+1.
/var/run/tor/control was no longer created and Tor throwing a warning.
Aug 03 21:56:34.000 [warn] Permissions on directory...
by adrelanos on 10:22PM - 03 Aug 16
changed 1 files with 8 additions and 3 deletions.

Updated anon-gw-anonymizer-config package testers package coming soon.
