Not a feature. Whonix does not ship source code which would add such restrictions and also currently do not ship a configuration file that would set a default resolution.

Likely an XFCE (version upgrade migration) bug.

What might (small chance) work if this is a version upgrade migration bug is to delete the XFCE config files (just the one for xfce4-display-settings). Instructions to wipe the whole XFCE settings folder and restore defaults:

  1. sort out a backup/restore
  2. Open a virtual console. - https://www.whonix.org/wiki/Desktop#Virtual_Consoles
  3. Stop XFCE.
    sudo systemct stop lightdm
  4. Trash folder ~/.config/xfce4
  5. Delete first boot skel copy done file.
    sudo rm /var/cache/anon-base-files/first-boot-skel.done
  6. Re-add Whonix XFCE config files.
    sudo /usr/lib/anon-base-files/first-boot-skel
  7. sudo systemctl restart lightdm

If that does not work, this XFCE bug has to be redirected to upstream, i.e. Debian and/or XFCE by you. (As per:

  • https://www.whonix.org/wiki/Free_Support_Principle
  • https://www.whonix.org/wiki/Reporting_Bugs#Linux_User_Experience_versus_Commercial_Operating_Systems)

A new download of Whonix 15 might also fix this (even though you don’t like that).
