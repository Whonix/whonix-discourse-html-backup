Qubes bug.

github.com/QubesOS/qubes-issues

qvm-template - strange interaction with dnf

opened 03:43PM - 08 Feb 22 UTC
unman
T: bug C: templates P: default diagnosed

[How to file a helpful issue](https://www.qubes-os.org/doc/issue-tracking/) ###… Qubes OS release 4.1 clean install and fully updated. ### Brief summary qvm-template has strange and confusing interaction with standard dnf tool in dom0. ### Steps to reproduce These are not steps, but separate use cases: 1. Enable qubes-templates-itl-testing repository in /etc/yum.repos.d/qubes-templates.repo 2. Install template with dnf install. 3. Install template with qvm-template tool. ### Expected behavior The output of qvm-template will be consistent with dnf configuration, and vice versa ### Actual behavior In Case 1: `qvm-template list` only shows available templates from qubes-templates-itl, although that repository is enabled. After #7170, it is possible to view the contents of qubes-templates-itl-testing , but this must be explicitly specified as an option to qvm-template. Case 2: Templates installed with dnf do not appear in the output of `qvm-template`. Case 3: Templates installed with qvm-template do not appear in the output of `dnf list installed`. All of these are unexpected and confusing.
