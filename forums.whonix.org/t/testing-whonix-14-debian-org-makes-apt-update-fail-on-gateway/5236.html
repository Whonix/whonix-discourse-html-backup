For virtualbox users, it matters whether you use apt or apt-get. apt-get is torrified. apt will try to communicate over clearnet, which Whonix-Gateway does not allow. Whonix-Gateway can optionally torrify your clearnet traffic but as you stated earlier, you have not changed the defaults: GATEWAY_TRANSPARENT_TCP=0; GATEWAY_TRANSPARENT_DNS=0. apt should simply fail.

Patrick May 23, 2018, 3:32am #17

By the way… Some addendum on technical details only.

Whonix-Gateway’s own traffic could be transparently torified (currently it has no system default networking, all is socksified pointing to Tor).

Whonix

Whonix-Gateways Own Traffic Transparent Proxy

Getting Transparent Proxying working for Whonix-Gateway's own traffic.

GATEWAY_TRANSPARENT_DNS alone is not enough to make DNS work on Whonix-Gateway. Please see whonix-firewall source code for the limited things that it does. User user, root user and others besides user clearnet would still be firewall restricted by default. You’d have to allow the root user (another whonix-firewall configuration option) to be able to connect and disable uwt for apt-get https://www.whonix.org/wiki/Stream_Isolation/Disable_Easy.

/etc/resolv.conf would have to be pointed to nameserver 127.0.0.1.

Whonix

Whonix-Gateway System DNS

Getting System DNS working on Whonix-Gateway
