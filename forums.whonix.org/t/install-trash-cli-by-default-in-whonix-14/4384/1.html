I have been thinking about this proposal for a while, and the following some of my thought:

  1. It seems that this is not a Whonix specific problem. Every Unix related tutorial online may suffer from this potential risk.
  2. It seems to violate the ‚Äúrule of least surprise‚Äù introduced in the art of Unix programming? Specifically,

‚Äú[t]he easiest programs to use are those that demand the least new learning from the user or, to put it another way, the easiest programs to use are those that most effectively connect to the user‚Äôs pre-existing knowledge‚Äù.

  1. trash-cli != rm According to the Github page of trash-cli:

‚ÄúAlthough the interface of trash-put seems to be compatible with rm, it has different semantics which will cause you problems. For example, while rm requires -R for deleting directories trash-put does not.‚Äù

User who does not read the trash-cli manual carefully and assume they are the same thing may make some mistakes?
