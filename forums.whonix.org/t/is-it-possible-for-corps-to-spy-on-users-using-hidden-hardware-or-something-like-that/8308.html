It will show you all the pointer and keyboard devices that your Xorg knows about. Note the ID of the device listed as ‚ÄúAT keyboard‚Äù and then run (as normal user!):

It should now start displaying the scancodes for all the keys you press on the keyboard. If it doesn‚Äôt, it means you used a wrong device ID.

Now, for the best, start another terminal window, and switch to root (e.g. using su, or sudo). Notice how the xinput running as user is able to sniff all your keystrokes, including root password (for su), and then all the keystrokes you enter in your root session.
