When I tested this before the host CPU serial number was not exposed in the guest and had different spoofed numbers.

What is not hidden however is the particular manufacturer of your CPU aka the VendorID. This can be forced but I don‚Äôt see the benefit. By default the CPU Family is chosen to be the one closest to the host‚Äôs to expose as much functionality to the guest as possible. This too can can be changed but it might prevent certain things like nested virtualization from working. I will not set these defaults unless you test that they don‚Äôt break nested virtualization.

More information on settings here:
https://wiki.openstack.org/wiki/LibvirtXMLCPUModel
