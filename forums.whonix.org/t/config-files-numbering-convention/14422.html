config files numbering convention

Development
Patrick August 9, 2022, 4:34pm #1

Not sure which numbering makes sense.

Draft:

  • 20 and lower for derivatives which Kicksecure / Whonix is based on such as Debian
  • 25 (lower than 30) in case the user configuration snippets are more useful to be configured before the Kicksecure / Whonix defaults
    • for example: /etc/permission-hardening.d/
  • 30 Kicksecure / Whonix default
    • for example: /etc/sdwdate.d
  • 40 config packages which change Kicksecure / Whonix defaults developed under Kicksecure / Whonix umbrella
    • for example: debug-misc
  • higher than 30 derivatives based on Kicksecure / Whonix
  • 50 and higher - user configured
    • mostly 50_user.conf

Numbers higher than 50 seem kinda “wasted” since not yet suggested for anyone and users probably sticking to 50_user.conf anyhow.
