In many situations, using popen is not good idea because it triggers parts of the shell that are directly affected by shellshock:

security.stackexchange.com

Are python's popen (and similar) functions affected by Shellshock?

shellshock, python
asked by Grzegorz Wierzowiecki on 02:27PM - 29 Sep 14 UTC
to my understanding this means that calls like:

os.popenx([‚Äòexecutable‚Äô, ‚Äòsome‚Äô, ‚Äòarguments‚Äô])

are safe because no shell is invoked, while commands of the form:

os.popenx(‚Äòexecutable some argument‚Äô)

are vulnerable because a shell will be used to interpret the string as a command line.

Also note that all popen functions are deprecated since python2.6 and should have been avoided since then. The subprocess module provides a much better interface to launching subprocesses which is shellshock safe by default. If you don‚Äôt pass the shell=True argument your program wont launch subshells and hence wont be affected by shellshock.
