I2P lacks stream isolation support

General Tor and Anonymity Talk
HulaHoop October 20, 2016, 8:36pm #1

Problem:

I2P does not have stream isolation support meaning that HTTP clients send the same X-I2P-Dest* headers to every eepsite visited - which allows adversaries running colluding eepsites to track user visits across sites.

Workaround:

New headers are generated when the I2P daemon is restarted so its recommended you restart I2P as often as possible to avoid profiling over long stretches of time.

Fix:

An I2P user is working on a plugin that uses SAM interface to retrieve sites over different connections. He said that it would be simple two day job.
