Hi there, I finally decided to tackle #169, about time. :) This is an initial… implementation. I've tested it with: ```sh sudo UPGRADE_SYSTEM='no' ./grml-debootstrap \ --debopt "--verbose" \ --arch arm64 \ --filesystem ext4 \ --hostname host \ --nopassword \ --nopackages \ --release bullseye \ --keep_src_list \ --verbose \ --vmfile \ --vmsize 15G \ --target ../debian-grml.raw ``` and I've been able to boot this image with qemu on an Apple Silicon based Mac (through UTM). Trouble is that I cannot really make sure I didn't break the amd64 builds, albeit I made every effort not to. I plan to test it a bit more but would appreciate an initial review at this stage. Thanks!

Which if merged would be super helpful for Whonix on Mac M1 (ARM) support. Even if not merged (soon), if there was a Whonix for Mac M1 (ARM) maintainer, I would accept carrying a mini fork of grml-debootstrap in derivative-maker help-steps folder.

Other than that, I am not aware of any contributor working on Whonix for Mac M1 (ARM) support. So no progress should be expected until that changes.
