No, it is ‚Äúfull system‚Äù. All processes are spawned off the init process so every single process (except kernel processes) are automatically confined. That includes ‚Äúall of /‚Äù.

It‚Äôs easily tested by running ps auxZ and you‚Äôll see every process is confined or you can try using ls on a non-permitted directory such as /boot.

No binaries have the ux or Ux rule which would execute them unconfined. They all have the pix rule which executes them under their own AppArmor profile and if it doesn‚Äôt exist, it falls back to the original init-systemd profile.
