      suppress: true

We can‚Äôt just suppress all of these events. For testing purposes, comment out all suppress: true. So it looks like this.

  events:
    CIRC:
      #suppress: true
    ORCONN:
      #suppress: true
    INFO:
      #suppress: true
    NOTICE:
      #suppress: true
    WARN:
      #suppress: true
    ERR:
      #suppress: true
    HS_DESC:
      #suppress: true
    HS_DESC_CONTENT:
      #suppress: true

The restart onion-grater on Whonix-Gateway.

sudo service onion-grater restart

So if you are able to use Bisq in this config. If yes, proceed with debugging below.

Comment in as many suppress: true as possible while Bisq is still fully connecting after Bisq restart, i.e. remove the # and sudo service onion-grater restart. As long as Bisq is saying ‚ÄúSynchronizing DAO‚Äù there is no need to proceed - that is the crux.
