Tor logs in Whonix discussion

Development
Patrick April 22, 2016, 9:43am #1

Debian Tor default:

Log notice file /var/log/tor/log

In Whonix 12 we are currently using:

Log notice file /var/log/tor/log
Log notice syslog
Log notice file /run/tor/log

Do we want to log to syslog?

Logging to /var/run/tor/log is useful for two reasons. It’s a shorter Tor log. Only since last reboot. And therefore it’s better for anondate to parse that one.

Currently all with loglevel notice. Example: [1] Currently there are unfortunately many very similar spammy messages. So it’s difficult to extract something useful from the log for both users and developers (such as when users post a log or during development).

Quote from Tor manual:

Recognized severity levels are debug, info, notice, warn, and err. We advise using “notice” in most cases, since anything more verbose may provide sensitive information to an attacker who obtains the logs.

Perhaps it would be useful to have an extra log that only contains messages of severity warn and err? I.e.

Log warn file /var/log/tor/log.warn_err
