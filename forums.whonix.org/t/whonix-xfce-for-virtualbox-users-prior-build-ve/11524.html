Whonix XFCE for VirtualBox users prior build version 15.0.1.7.3: Storage Controller Setting Change Required (Failed to open a session for the virtual machine) (NS_ERROR_FAILURE (0x80004005))

News
important-news
Patrick April 27, 2021, 5:40pm #1

Affected Versions

  • Whonix for VirtualBox build version lower than 15.0.1.7.3 → Virtual (VM) Storage Controller settings change required.
  • Whonix for VirtualBox build version 15.0.1.7.3 (and higher) → no change required since fix is included.

Symptom

Failed to open a session for the virtual machine

NS_ERROR_FAILURE (0x80004005)

Impact

Whonix VirtualBox does not start until setting is changed.

Solution

For instructions, see wiki:
Failed to open a session for the virtual machine

VirtualBox

This issue was introduced between VirtualBox 6.1.18 and 6.1.20.
(virtualbox.org skipped, never released VirtualBox 6.1.19.)

Therefore before (or after) upgrading recommended VirtualBox version to 6.1.20 the settings change required.

Wiki page recommended VirtualBox version takes precedence for version recommendations.
