Algernon October 10, 2018, 9:52pm #2

  1. For more output you can try to launch the tor browser directly from the folder in verbose mode i.e. “./start-tor-browser -v” . Also take a look at the log, i.e. “sudo journalctl”, or use “sudo journalctl -f” and then try to start the browser.

  2. Normally just starting the two VMs should be sufficient and you don’t need to setup anything else. The Tor browser problem is likely not related to any connectivity issue. You can also use whonixcheck, any other command which needs internet access or wireshark to see if there are any network problems.

  3. Should be. In case you don’t get any connection to Tor you should test it without a VPN.
