Hi all,

I’ve just uploaded new Whonix 10 templates for both R2 and R3.0rc1.
Actually there was no Whonix templates for R3.0rc1 earlier, so those are
the first one.

Major changes:

  1. Upgrade Whonix 9 to Whonix 10
  2. Rename templates to shorter names. New names:
  • whonix-gw-experimental
  • whonix-ws
  • whonix-ws-gnome

Additionally I’ve uploaded Qubes Builder configuration suitable to make
the same build on our own:

GitHub

QubesOS/qubes-template-configs

qubes-template-configs - Qubes Builder configs for template builds

To install the templates, issue:
sudo qubes-dom0-update --enablerepo=qubes-templates-community
qubes-template-whonix-gw-experimental

sudo qubes-dom0-update --enablerepo=qubes-templates-community
qubes-template-whonix-ws
(or -gnome flavor)

Details on Whonix website: https://www.whonix.org/wiki/Qubes

Best Regards,

You can do it either way. If building Qubes, you may want to use the setup program I created. There is a bug though where you need to have a depend installed first (python-sh) since Marek took it out of libs directory in flavor of installed package.

So, to build; install python-sh, run setup; select options; make build-deps; make get-sources; run setup again… second time will have the options for Whonix vm’s to build as well.

Otherwise, just follow the instructions in OP or Wiki to install Whonix after Qubes is installed. You can also install Qubes from ISO DVD or USB to save hassle of building.

Good luck!
