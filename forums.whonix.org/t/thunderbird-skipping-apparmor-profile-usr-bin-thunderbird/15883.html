Confirmed.

File /etc/apparmor.d/usr.bin.thunderbird is owned by package thunderbird by packages.debian.org.

dpkg -S /etc/apparmor.d/usr.bin.thunderbird

thunderbird: /etc/apparmor.d/usr.bin.thunderbird

And that Debian package has some rather strange behavior:

Debian Bug report logs - #884191
Please do not disable AppArmor silently and prevent its re-activation

In other words, Debian does not enable that AppArmor profile by default and the way to enable the AppArmor profile by default is rather unexpected.

Package GitHub - Kicksecure/apparmor-profile-thunderbird: AppArmor profile for Thunderbird - https://www.kicksecure.com/wiki/AppArmor - for better security (hardening). is currently only shipping additions:
apparmor-profile-thunderbird/usr.bin.thunderbird at master Â· Kicksecure/apparmor-profile-thunderbird Â· GitHub
