Patrick August 22, 2015, 7:58pm #2

It’s not a wanted feature. It’s either a bug in Whonix, the virtualizer or the host operating system. Therefore there is no obvious way to disable it. Best that can be done is asking questions back and forth. Hoping that way the cause can be debugged. So a workaround can be suggested as quick fix and so the bug might be fixed with an update.

Which virtualizer?

Which host operating system?

X, the graphical environment crashes and you’re left in the text console?

If you use your computer actively without inactivity, then Whonix-Gateway works fine for an unlimited amount of time?

Did you try to disable the screen saver on the host if it still happens then during inactivity? (I am not suggesting this as a permanent solution. For debugging only.)

Would be worth keeping a process monitor running. And “iotop -a”. Eventually monitoring things from tty0. Also check syslog. Do you know about this stuff?

As long as the desktop is running, you can start…

Then sort by CPU usage. Or memory. Keep looking what uses most or non-ordinary high amounts by comparison.

Also…

Install some tools.

(right control is default)

Keep looking. See if any processes use and ridiculous amount >= 50% or more.

Also, keep watching syslog.

If it shows anything interesting. Or post the syslog here.
