On a freshly installed and working Whonix-Workstation, I follow the VPN Firewall README instructions from here:

GitHub

adrelanos/vpn-firewall

vpn-firewall - Leak Protection (Fail Safe Mechanism) for (Open)VPN

sudo cp ./usr/bin/vpnfirewall /usr/bin/

sudo nano /usr/bin/vpnfirewall

The VPNBook German VPN IP in the OpenVPN config is: “remote 178.162.193.233 443”

So I change the vpnfirewall config to:

VPN_SERVERS=“178.162.193.233”

sudo /usr/bin/vpnfirewall

OK: Loading VPN firewall…
OK: The firewall should not show any messages,
OK: besides output beginning with prefix OK:…
OK: VPN firewall loaded.

Everything looks ok, so I run my OpenVPN command:

sudo /usr/sbin/openvpn --config ~/*.ovpn

That loads normally and seems to connect successfully too “Initialization Sequence Completed”.

I just can’t send any internet traffic through the VPN at this point like I could without the VPN Firewall loaded.

With VPN Firewall = No VPN internet connection.
Without VPN Firewall = Yes VPN internet connection.
