Whonix XFCE for VirtualBox users: RAM increase required

News
important-news
Patrick February 24, 2020, 6:44am #1
  • Releases of Whonix-Gateway XFCE prior build version 15.0.0.8.9: Virtual (VM) RAM increase to 1280 MB RAM advised. [1]

Otherwise APT upgrades might freeze during module building (VirtualBox guest additions) as well as kernel header package upgrade. [2]

To change RAM setting:

  1. Power off the virtual machine.
  2. Open VirtualBox.
  3. Go to VirtualBox machine settings â†’ System â†’ Base Memory â†’ Increase RAM by moving the slider or type 1280.
  4. Ignore message about invalid graphics controller VMSVGA because, see this.
  5. Click OK.
  6. Power on the virtual machine again.
  7. Done.
  • Whonix-Workstation XFCE: Already comes with sufficient RAM by default. No changes required.
  • Whonix 15.0.0.8.9 and above already comes with sufficient RAM by default. No changes required.
  • Whonix CLI: Unaffected. No changes required. -> Whonix Gateway CLI-15.0.1.5.4 - "VirtualBox Guest Additions: Building the modules for kernel 4.19.0-16-amd64" freeze
  • Whonix KVM: Unaffected. No changes required.

[1] 1280 MB RAM is sufficient. Using more would be OK too. Such as 2048 MB RAM would be ideal since then no swap file would be created.

[2] These are the trigger but not the root cause of this bug. The root cause might be a kernel bug where system becomes unresponsive when swapping that only occurs in VirtualBox that probably will not be fixed soon.
