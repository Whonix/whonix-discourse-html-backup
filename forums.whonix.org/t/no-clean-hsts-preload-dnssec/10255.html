TNT_BOM_BOM:

HSTS policies must not be transmitted over insecure channels.

I am inclined to ignore this unless someone has an argument why it is an actual issue.
Reason: no actual security issue for anyone.
In case of:

  • in case of no MITM: browser will ignore the HSTS header
  • in case of MITM: the MITM can inject whatever malicious HSTS header
    since this is over an unencrypted/unauthetnicated https connection (this is about the http version (non-TLS) (stub to redirect to https / TLS)).

Also‚Ä¶

  • In case browser honors HSTS preload list: this wouldn‚Äôt happen anyhow
  • In case browser ignores HSTS preload list: then whatever the clearnet version headers are cannot help anyhow

It might be counter to some standard but I don‚Äôt see how it helps. It would complicate server nginx config for minimal gain (namely making a test website happy).

TNT_BOM_BOM:

Redirection from HTTP to HTTPS not to the same host:

This seems a bit pointless to me to redirect http://whonix.org (which we don‚Äôt use - whonix.org currently doesn‚Äôt use the non-subdomain, main domian) except for redirect to https://whonix.org to only then redirect to https://www.whonix.org. That would be a double redirect and make the website slower for those who just type whonix.org. Other test websites focused on website performance would complain about the double redirect.

TNT_BOM_BOM:

All nameservers in the delegation have IPv4 addresses in the same AS

What this means and how to fix it if necessary:

You‚Äôre getting this warning because the nameservers are all in the same block and if that single route goes offline for some reason, all your nameservers go down. It‚Äôs generally best to spread these out geographically to minimize your exposure to localized events.

They‚Äôre just looking out for you here. Typically you should have 3-4 different nameservers on different backbone providers in different regions so that no single failure, even at the provider level, can take them all down.

You have the domain name, TTL, protocol, record type. Then ‚Äúmname‚Äù which should be the name of the PRIMARY domain name server, followed by ‚Äúrname‚Äù which is the zone admin email (using a . instead of an @) and then serial #, refresh, retry, expire, and a default Time-To-Live for records.

Your domains mname should be the primary name server.
