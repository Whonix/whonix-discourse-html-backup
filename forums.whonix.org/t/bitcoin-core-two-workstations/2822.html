Temple:

I’ve been reading through the wiki and if I’m correct I can run one Gateway and two Workstations at the same time by changing the internal ip of one to 10.152.152.12

Whonix

Multiple Whonix-Workstations

Compartmentalization. Better separation of different tasks and/or pseudonyms by using multiple Whonix-Workstations.

Temple:

if I was going to run a hidden service which also connects to a bitcoin core node i’d host the bitcoin core node on a different server for safety

That sentence is quite difficult for me to understand.

I assume it means something like this “If I was going to run a hidden service webserver which also connects to a bitcoin core node i’d host the bitcoin core node on a different server for safety”

In this scenario you might have have to make sure the connections between these two servers is encrypted and authenticated. If hidden services talk to each other, it would be encrypted. (details: https://www.whonix.org/wiki/Hidden_Services#Hidden_Service_Authentication)

Depends on the platform. For Qubes-Whonix, you need to allow communication between the VMs as per Qubes documentation.

For Non-Qubes-Whonix these can communicate without any changes required.

( https://www.whonix.org/wiki/Connections_between_Whonix-Gateway_and_Whonix-Workstation )
