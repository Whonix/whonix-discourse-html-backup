Reduce numbers of grub live-mode boot menu entries Development
whonix startup confusing grub live-mode options: whenever there is a new linux kernel version installed it will add up to the live-mode list instead of choosing the latest version. the list going to be like this: start whonix advanced... live-mode kernel x live-mode kernel y live-mode kernel z facially and practically not great. Solution: disable GRUB_DISABLE_SUBMENU="y" then its going to be: start whonix advanced... Live mode advanced... way much better.
