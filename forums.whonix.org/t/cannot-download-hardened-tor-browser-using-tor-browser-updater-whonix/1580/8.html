Hm. How awful. They changed the download link scheme again. There is no way to foresee such issues. (Unless programming some intelligence that guesses this.)

There is a workaround. You can set tb-updater’s TB_LANG variable to ALL. (case sensitive, make sure to write ALL with all upper case.) This is one method to do this:

update-torbrowser --language ALL

There are also other ways set tb-updater’s TB_LANG variable (permanently) (by config file, etc.):
https://www.whonix.org/wiki/Language#Tor_Browser

But when doing so, you will no longer be able to download the normal versions. Before being able to do that, you need to undo that setting.

(Not Whonix specific. Moving to Whonix support forum.)

Non-Qubes-Whonix:

  • Not easy…
  • Install and boot kernel linux-image-amd64. Or set environment variable ‘export ARCH_DOWNLOAD=linux64’. Or not. Even if installed by tb-updater or manually. It’s a dead end. Won’t work in Non-Qubes-Whonix. At least not with the Default/Download version. Because the architecture is 32 bit.
  • Experiment with multiarch. Try installing 64 bit packages also. Looks like an inefficient solution.
  • Build Non-Qubes-Whonix 64 bit from source code.

Quote https://blog.torproject.org/blog/tor-browser-55a4-hardened-released

For now this is for Linux 64bit systems only but we are thinking about supporting OS X and Windows in the future as well.
Doesn't look like Linux 32 bit hardened TBB downloads will be provided anytime soon.

EDIT:
It’s also very unlikely, that I will be producing downloadable 64 bit versions of Non-Qubes-Whonix. Too much maintenance effort. I try balance it by providing 32 bit Non-Qubes-Whonix for most/lower end systemd and providing 64 bit Qubes-Whonix for higher end systems.

Non-Qubes-Whonix 14 will be 64 bit.
(full discussion on 32 vs 64 bit: https://forums.whonix.org/t/state-of-offical-64-bit-builds)
