Whonix/grub-output-verbose

Enables verbose output during boot - For better usability, so it doesn't look like boot hangs on slow systems and to ease debugging in case of issues. - Whonix/grub-output-verbose

It used to be considered a usability feature. And I still find it much more useful to show verbose boot messages than a static splash screen. It also allows to easier debug boot failures.

However, for security reasons it needs to be disabled by default. Some references:

  • Untrusted Root - improve Security by Restricting Root
  • Kernel Hardening

  • Hide Kernel Symbols for Better Security vs Reproducible Builds
