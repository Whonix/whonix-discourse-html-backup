I think it makes sense to continue the topic in this thread.

The current state is that when you run dpkg-reconfigure i2p in the Whonix Template (whonix-ws-16-clone-1), it does not create the configuration files under /var/lib/i2p/. I don‚Äôt currently see a suitable way to do this, which is the problem. The instructions in the wiki do not work after step 3.

Additional question: Why should should someone securely download the key to the Whonix-Workstation as mentioned in step 1 instead of only downloading it to Whonix Template? Or is this necessary for every case?

Current alternative is after doing steps 1-3 in the Whonix Template (whonix-ws-16-clone-1):

  1. Shutdown Whonix Template (whonix-ws-16-clone-1)
  2. Start Whonix-Workstation AppVM (anon-whonix-I2P) based on Whonix Template (whonix-ws-16-clone-1)
    Following steps have to be done on every restart of the AppVM due to non-persistance.
  3. Run dpkg-reconfigure i2p which also start i2p and creates configuration files in /var/lib/i2p/
  4. Stop I2P service with sudo systemctl stop i2p
  5. Use sudoedit /var/lib/i2p/i2p-config/clients.config.d/00-net.i2p.router.web.RouterConsoleRunner-clients.config and change 127.0.01 to 127.0.0.2.
  6. Start I2P service with sudo systemctl start i2p
  7. Optional: Adjust I2P Router configuration

The following step only needs to performed once and after TB updates:
8. Configure TB for usage with I2P

TB can be used to access I2P pages. TB can not be used to access onion addreses (Outproxy not found). TB can be used to access clearnet. Should we avoid this due to security considerations and why? Any other things to consider with this setup in terms of security or anonymity?

Regarding step 7: Do we have to configure router.config.anondist in /usr/share/i2p/ or /var/lib/i2p/i2p-config/ to be effective?

Would appreciate any solution to be able to setup I2P service and configuration files in Whonix Template (whonix-ws-16-clone-1).
