Maybe the profile folder change during the re-branding is related (?). See:

https://www.debian.org/security/2017/dsa-3832

With this update, the Icedove packages are de-branded back to the official Mozilla branding. With the removing of the Debian branding the packages are also renamed back to the official names used by Mozilla.

The Thunderbird package is using a different default profile folder, the default profile folder is now ‚Äò$(HOME)/.thunderbird‚Äô. The users profile folder, that was used in Icedove, will get migrated to the new profile folder on the first start, that can take a little bit more time.
