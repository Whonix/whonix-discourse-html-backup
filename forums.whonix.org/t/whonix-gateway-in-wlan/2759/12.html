entr0py July 19, 2016, 4:52pm #5
Syms:

*1. Install Whonix-Gateway on Virtuablox

In a Physical Isolation build, the recommendation is to install Whonix-Gateway directly on your hardware (bare-metal). You increase your attack surface by adding an additional OS and Virtualizer to the setup without gaining any real benefits.
https://www.whonix.org/wiki/Dev/Build_Documentation/Physical_Isolation#How_To_Install_Whonix-Gateway_in_a_VM_.28UNTESTED_.2F_NOT_RECOMMENDED.29

Syms:

*2. In Whonix-Gateway install and configure VPN

Syms:

*3. Connect from workstation to this VPN and then all trafiic will be through TOR yes ?

Please be sure you are using the correct terminology. VPN is not the same as virtual LAN.

Also:

  • Keep in mind that Physical Isolation is not (well) supported and is intended for advanced users who can troubleshoot on their own. Please consider @Ego’s advice about using Qubes since it may be just as (or more) leak-proof than physical isolation. http://www.invisiblethingslab.com/resources/2014/Software_compartmentalization_vs_physical_separation.pdf

  • Just noticed subject line of post. If your intention is to connect the 2 computers via Wireless networking, you’re probably better off using Qubes. Using wireless may necessitate using a VPN with a fail-safe: see https://www.whonix.org/wiki/Connections_between_Whonix-Gateway_and_Whonix-Workstation#Basics
