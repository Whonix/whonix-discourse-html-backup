Patrick November 6, 2018, 2:23pm #4

In Qubes R4 and above: Networking (NetVM) of TemplateVM (Whonix, Debian, Fedora, probably any) should be set to none is ok.

Reminder: In Qubes R4 and above the NetVM setting of TemplateVMs should be set to None .

Since TemplateVMs in Qubes R4 and above are supposed to be upgraded through qrexec based Qubes updates proxy.

  • https://www.qubes-os.org/doc/software-update-vm/#technical-details-r40
  • implement qrexec based updates proxy
sudo qubesctl state.sls qvm.whonix-ws-dvm
