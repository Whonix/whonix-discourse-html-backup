It won‚Äôt install any files to /usr/bin. I think the confusion comes from how Debian handles config files in /etc.

As per debian/apparmor-profile-torbrowser.install the package installs only two important files

It takes home.user.tor-browser_en-US.Browser.firefox and installs it to /etc/apparmor.d/ and it takes local/home.user.tor-browser_en-US.Browser.firefox and installs it to /etc/apparmor.d/local/. (Plus some ‚Äúless important files‚Äù in /usr/share/doc/apparmor-profile-torbrowser/.)

When you manually made changes to /etc/apparmor.d/home.user.tor-browser_en-US.Browser.firefox, then Debian will remember that and not overwrite your config file again. Which means, when you install the package, it won‚Äôt overwrite this file. To get rid of this behaviors, you can ‚Äúsudo apt-get purge apparmor-profile-torbrowser‚Äù (and eventually manually delete that file beforehand). Then home.user.tor-browser_en-US.Browser.firefox can be found in /etc/apparmor.d/ after installing the package with ‚Äúdpkg -i‚Äù.
