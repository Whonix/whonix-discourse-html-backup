MAC address randomization?

Support
hunixtalk December 21, 2021, 3:51pm #1

Does randomizing MAC address further defend de-anonymizing? Suppose there’s a Firefox bug that allows JavaScript to read MAC address (and further, router MAC like in ip n), all your web browsing is then linked.
Maybe it’s not practical to do this on the hypervisor level, how about VM MAC spoofing? Is it a good idea to change Gateway and Workstation MAC address at times?

As for MAC address inside VMs…

Using Multiple Whonix-Workstation ™ is generally recommended.

If such a bug allows to read the MAC address of the VM, what other things should be presumed readable from remote (such as Firefox when having a bug)? I am asking, because then the following would apply:

Whonix – 2 Dec 21

VM Fingerprinting

Linking two or more locally compromised Virtual Machines (VMs) to the same pseudonym.

See also:

MAC address anonymization Development
I would like to raise community awareness on MAC address anonymization. Tails project actually documents various threats at https://tails.boum.org/contribute/design/MAC_address Most concerning to me actually is “Active probe fingerprinting” documented at https://tails.boum.org/contribute/design/MAC_address/#index7h1 While using Wifi is generally recommended against, I guess people use it and from my perspective we should closely follow what Tails is going to release here (currently in testing)…
