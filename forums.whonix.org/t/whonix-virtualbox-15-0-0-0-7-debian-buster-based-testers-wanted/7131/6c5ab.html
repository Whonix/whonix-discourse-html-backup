BTW, should we move from apt-get to apt?

Difference Between apt and apt-get Explained

Explaining how apt command is similar yet different than apt-get and why you should be using apt instead of apt-get.

So it seems apt is a little more user-friendly (in general) and is in Debian by default.

Annoying that apt, apt-get and aptitude are all slightly different with different pros vs. cons, but at least we have choice for different situations. bug: not all files form /etc/skel are copied to /home/user Development
Not all files from /etc/skel are currently copied to /home/user. This is because package anon-base-files which creates user user and thereby runs useradd which copies /etc/skel is installed before usability-misc. In result folder /home/user/Downloads is no longer created by default in Whonix 14. (A long standing, non-critical, non-release-blocking bug.)
