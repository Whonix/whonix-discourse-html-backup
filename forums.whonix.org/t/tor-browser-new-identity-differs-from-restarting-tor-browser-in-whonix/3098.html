I‚Äôve been researching this topic a bit and I‚Äôm going to disagree with some things that @Patrick said (!!).

winibub:

I realise this might be a tor issue, but I‚Äôm not sure.

Right. This is a Tor Browser issue. Whonix is not involved here. Shutting down and restarting is specific to Whonix (per HulaHoop‚Äôs comment.) Tor Browser has built-in circuit isolation features. A more descriptive title would be ‚ÄúTor Browser circuit isolation‚Äù.

winibub:

shutting down tor browser and then opening it again would do what the New identity button does.

Other than clearing the application-level browser data, I do not know exactly what closing Tor Browser entails. However, the Torbutton New Identity function is well documented here: https://www.torproject.org/projects/torbrowser/design/#new-identity

One thing that New Identity does that closing Tor Browser might not do is this‚Ä¶

After the state is cleared, we then close all remaining HTTP keep-alive connections and then send the NEWNYM signal to the Tor control port to cause a new circuit to be created.

This would explain why IP results differ between closing TB and issuing New Identity.

winibub:

the Ip stays the same.

This is probabilistic evidence so make sure you run the test enough times to be sure. Even with ‚ÄúNew Identity‚Äù, you‚Äôre IP address might stay the same: https://tor.stackexchange.com/questions/230/when-i-click-new-identity-why-do-i-sometimes-end-up-with-the-same-exit-relay

winibub:

Does this mean that all my web activity is now linked to one single profile ‚Äútor identity‚Äù / circuit (for each starting up of whonix)? Is there anyway that this is not the case?

Assuming my edit of your question is what you intended, your web browsing was never (since TBB v4.5a1) limited to a single circuit. There are two ways that your browsing activities are isolated from each other:

1. Over time:
https://stem.torproject.org/faq.html#how-do-i-request-a-new-identity-from-tor

Tor periodically creates new circuits. When a circuit is used it becomes dirty, and after ten minutes new connections will not use it. When all of the connections using an expired circuit are done the circuit is closed.

2. By first-party domain name
Several places in the Whonix wiki (including https://www.whonix.org/wiki/Tor_Browser) refer to Tor Browser Tab Isolation. AFAICT this is incorrect. https://trac.torproject.org/projects/tor/ticket/3455 explains that streams are isolated by SOCKSauth and that SOCKS username is a function of the base url first-party domain name. This can be seen by opening an arbitrary number of tabs and browsing to check.torproject.org. All of the tabs will show the same IP address. In order for multiple connections to the same website to use a different circuit, a separate instance of Tor Browser is required.

EDIT: Also of interest to OP, Tor Browser Bundle (TBB) new circuit versus new identity.

Here‚Äôs what I think you‚Äôre asking:

Within a 10 minute timeframe,

  1. You connected to gmail.com as Joe.
  2. You restarted Tor Browser.
  3. You connected to gmail.com as Mary.

In this case, both Joe and Mary would have connected to gmail.com over the same Tor circuit.

You can watch this process yourself by installing onioncircuits. It‚Äôs a great tool to learn about stream isolation.
