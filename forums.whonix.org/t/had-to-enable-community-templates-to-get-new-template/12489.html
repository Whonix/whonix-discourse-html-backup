Talked to Marek at 34c3. We concluded the following. * Installing Whonix manu‚Ä¶ally using `qubes-dom0-update` got too difficult since too many manual modifications of dom0 would be required. * Most users shall no longer install Whonix using `qubes-dom0-update`. * Most users shall install Whonix using Qubes `salt`. * Most users shall no longer update using `qubes-dom0-update`. * Should use `sudo qubesctl --templates pkg.upgrade` instead. * We need `salt` commands for installation of arch linux and others * Modify `qubes-dom0-update` to tell the user it is recommended to use `salt` instead.
